# 需求说明

> 功能边界,用来判断"这个功能到底算不算完成"。更新于 2026-06-21。

## 总目标

一个**在手机上独立运行**、能**控制手机一切**、具备**分层记忆**、可**后台常驻**的自主 agent,
能帮用户在真实 app 里完成多步任务(发消息、点外卖等)。运行时借鉴 OpenAI Codex。

## ① 形态 —— ✅ 已达成

- 主循环在手机 Termux 内跑(on-device),用 `su` 拿 screencap/uiautomator/input/am。
- 不依赖常驻 PC(PC 只作开发/调试通道)。
- 大脑走云端 API(非本地大模型)。

## ② 操作能力 —— ✅ 大部分达成

- 能截图、读 UI 树、点击、滑动、输入(含中文)、跨应用切换。
- 规则:模型只能发 schema 约束的工具(tap / tap_xy / swipe / type / paste / submit / key / launch_app / wait / remember / finish / ask_user)。
- **完成判定**:给一个自然语言目标,agent 能在真实 app 里达成并 `finish`,rollout 有完整记录。

## ③ 后台运行 —— ❌ 待建

- 需求:headless 常驻服务,不用每次手敲命令;到点/收到指令就自动干活。
- **完成判定**:关掉终端后 agent 仍能在后台按触发执行任务。

## ④ 真实世界任务(如点外卖)—— 🔶 部分

- 需求:多步、跨应用、涉及真实后果(下单、支付)的任务。
- 规则:
  - 跨应用交接走 intent 优先 → 剪贴板 → 读取-重输。
  - **危险动作(发送/支付/删除/安装)必须过审批**;后台场景走异步推送审批。
  - **agent 永不自动花钱**——支付一定要用户确认。
- **完成判定**:能完成一个跨 ≥2 个 app 的真实任务,支付步骤被审批拦截并等用户确认。

## ⑤ 分层记忆(模仿 Codex)—— ❌ 核心待建

五层(详见 [data-model.md](data-model.md) 与 notes):
1. 工作记忆(当前 turn)—— ✅ 有
2. 会话记忆(rollout,可回放)—— ✅ 有
3. 摘要/压缩 —— ❌
4. 长期记忆(跨会话事实,检索注入)—— ❌ 核心
5. 程序记忆(成功任务固化成 flow)—— ❌
- **完成判定**:第二次做同类任务时,agent 能用上上次记住的事实(如地址),不用重问。

## ⑥ 触发方式 —— ⬜ 待定

候选:定时任务 / 远程消息(自己给自己发指令)/ 语音。**需用户拍板。**

## 非目标(v0)

- 完全离线本地大模型;越权/对抗检测规避;批量多设备。
